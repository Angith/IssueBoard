# Implementation Plan: Issue Board Core

**Branch**: `001-issue-board-core` | **Date**: 2026-04-26 | **Spec**: [specs/001-issue-board-core/spec.md]
**Input**: Feature specification for a web application called issueBoard.

## Summary

Build a system for managing a GitHub repository inventory and viewing issues grouped by labels. The system will use a monorepo architecture with a Go backend and a React/Next.js frontend. It will feature GitHub OAuth via Supabase, PostgreSQL for persistence and caching, and direct integration with the GitHub REST API.

## Technical Context

**Language/Version**: Go 1.21+ (Backend), React/Next.js (Frontend)
**Primary Dependencies**: Supabase (Auth/DB), GitHub REST API (Octokit-style or custom client)
**Storage**: PostgreSQL (Docker for local, Supabase for production)
**Testing**: Go `testing` package (70% coverage goal), ESLint/Prettier (Frontend)
**Target Platform**: Web (modern browsers)
**Project Type**: Web application (monorepo)
**Performance Goals**: < 2s login/inventory view, < 500ms cached issue board load
**Constraints**: READ-ONLY interaction for issues/repos, strictly separated FE/BE, environment-based DB config.
